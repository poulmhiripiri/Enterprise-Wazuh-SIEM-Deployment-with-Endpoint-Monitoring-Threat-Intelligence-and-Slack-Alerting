# 02 - Agent Onboarding

## Objective

Onboard Windows and Linux systems into Wazuh so endpoint events can be collected and analysed centrally.

## Windows Agent Onboarding

The Windows endpoint was enrolled using the Wazuh dashboard agent deployment workflow. After installation and service startup, the endpoint reported successfully into the Wazuh dashboard.

## Linux Agent Onboarding

The Linux endpoint was enrolled using the Wazuh-provided installation command for the relevant operating system and architecture. After enrollment, the Linux host appeared in the dashboard and was available for security monitoring.

## Operational Value

Agent onboarding gives SMEs visibility across servers and user endpoints. This supports detection of authentication failures, malware events, file integrity changes, vulnerabilities, and suspicious endpoint behaviour.
